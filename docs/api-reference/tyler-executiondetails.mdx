## Overview

<Note>
**Important**: The `ExecutionDetails` class has been removed from Tyler to eliminate redundancy. All execution telemetry is now available directly through the `Thread` and `Message` objects.
</Note>

## Migration Guide

The information previously available in `ExecutionDetails` can now be accessed through Thread utility methods and Message metrics:

### Token Usage

**Before:**
```python
result = await agent.go(thread)
print(f"Total tokens: {result.execution.total_tokens}")
```

**After:**
```python
result = await agent.go(thread)
token_stats = result.thread.get_total_tokens()
print(f"Total tokens: {token_stats['overall']['total_tokens']}")

# You can also get per-model breakdown:
print(f"By model: {token_stats['by_model']}")
```

### Execution Duration

**Before:**
```python
print(f"Duration: {result.execution.duration_ms}ms")
```

**After:**
```python
if result.new_messages:
    start_time = min(msg.timestamp for msg in result.new_messages)
    end_time = max(msg.timestamp for msg in result.new_messages)
    duration_ms = (end_time - start_time).total_seconds() * 1000
    print(f"Duration: {duration_ms:.0f}ms")
```

### Tool Usage

**Before:**
```python
for tool_call in result.execution.tool_calls:
    print(f"Tool: {tool_call.tool_name}")
    print(f"Success: {tool_call.success}")
```

**After:**
```python
tool_usage = result.thread.get_tool_usage()
print(f"Total tool calls: {tool_usage['total_calls']}")
for tool_name, count in tool_usage['tools'].items():
    print(f"{tool_name}: {count} calls")
```

### Message-Level Metrics

Each message now contains its own metrics:

```python
for message in result.new_messages:
    if message.metrics:
        print(f"Model: {message.metrics['model']}")
        print(f"Latency: {message.metrics['timing']['latency']}ms")
        print(f"Tokens: {message.metrics['usage']['total_tokens']}")
```

## Available Thread Methods

The Thread object provides several utility methods for accessing execution information:

- `thread.get_total_tokens()` - Get token usage across all messages
- `thread.get_tool_usage()` - Get tool call counts
- `thread.get_model_usage()` - Get usage statistics per model
- `thread.get_message_timing_stats()` - Get timing statistics
- `thread.get_message_counts()` - Get message counts by role

## Streaming Events

For real-time execution monitoring, use streaming mode which still provides `ExecutionEvent` objects:

```python
async for event in agent.go(thread, stream=True):
    if event.type == EventType.LLM_RESPONSE:
        print(f"Tokens: {event.data['tokens']}")
    elif event.type == EventType.TOOL_RESULT:
        print(f"Tool result: {event.data['result']}")
```

## Why This Change?

The `ExecutionDetails` class was removed because:
1. It duplicated information already stored in messages
2. Thread utility methods provide the same aggregations
3. Weave integration offers detailed debugging when needed
4. Simpler API with less redundancy
